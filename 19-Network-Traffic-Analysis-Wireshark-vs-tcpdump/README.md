# Network Traffic Analysis – Wireshark vs. tcpdump

## Overview

This project involved comparing two widely used network protocol analysis tools: **Wireshark** and **tcpdump**.

The objective was to evaluate their capabilities, identify similarities and differences, and understand how each tool can be used to capture, inspect, and analyze network traffic for cybersecurity monitoring, troubleshooting, and incident investigations.

---

## Objective

The purpose of this project was to:

- Research the functionality of Wireshark and tcpdump
- Compare graphical and command-line packet analysis tools
- Evaluate their use in security monitoring and troubleshooting
- Identify strengths and limitations of each solution
- Understand how network analysts use packet capture tools during investigations

## Key Findings

### Similarities

Both Wireshark and tcpdump:

- Are open-source network protocol analyzers
- Capture and inspect network traffic
- Support packet filtering and traffic analysis
- Assist with troubleshooting and security investigations
- Provide visibility into network communications

### Wireshark Characteristics

Wireshark offers:

- Graphical user interface (GUI)
- Advanced protocol decoding
- Detailed packet inspection
- Visualization and reporting capabilities
- User-friendly analysis features

Wireshark is particularly useful for:

- Security investigations
- Network troubleshooting
- Protocol analysis
- Training and learning environments

### tcpdump Characteristics

tcpdump offers:

- Command-line interface (CLI)
- Lightweight operation
- Low resource consumption
- Fast packet capture capabilities
- Suitability for remote systems and servers

tcpdump is particularly useful for:

- Linux environments
- Production servers
- Remote administration
- Resource-constrained systems

## Comparison Summary

| Feature | Wireshark | tcpdump |
|----------|----------|----------|
| Interface | Graphical (GUI) | Command-line (CLI) |
| Ease of Use | Beginner-friendly | Requires command-line knowledge |
| Packet Analysis | Extensive visual analysis | Text-based analysis |
| Resource Usage | Higher | Lower |
| Remote Server Usage | Less common | Highly suitable |
| Protocol Decoding | Advanced | Basic to intermediate |
| Visualization | Strong | Limited |
| Automation | Limited | Strong scripting support |

## Lessons Learned

### Different Tools Can Achieve Similar Goals

The project demonstrated that multiple cybersecurity tools may accomplish similar objectives while supporting different operational requirements.

### GUI Tools Simplify Analysis

Wireshark simplifies packet inspection through:

- Visual packet decoding
- Protocol breakdowns
- Filtering capabilities
- Interactive analysis

### Command-Line Tools Remain Essential

tcpdump is often preferred because of:

- Efficiency
- Portability
- Low resource consumption
- Ease of deployment on servers

### Broader Analyst Capability

Understanding both tools improves an analyst’s ability to investigate network-related incidents across different environments and infrastructures.
