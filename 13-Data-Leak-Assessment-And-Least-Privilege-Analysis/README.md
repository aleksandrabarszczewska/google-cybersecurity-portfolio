# Data Leak Risk Assessment and Least Privilege Analysis

## Overview

This project presents a security assessment conducted following a data leakage incident at an educational technology company.

The investigation focused on identifying the root cause of unauthorized data exposure, evaluating access control weaknesses, and recommending security controls aligned with the Principle of Least Privilege and NIST guidance.

## Scenario

The organization experienced accidental exposure of confidential internal documents due to excessive user permissions and insufficient access governance controls.

The assessment was conducted to determine contributing factors, evaluate risk, and develop recommendations to prevent similar incidents in the future.

## Project Objectives

The project focused on:

- Data leak investigation
- Access control review
- Least privilege analysis
- Privacy protection

## Assessment Activities

The investigation included:

- Reviewing incident details
- Identifying factors contributing to unauthorized data exposure
- Evaluating access control weaknesses
- Assessing security controls using the NIST Cybersecurity Framework (CSF)
- Reviewing NIST SP 800-53 AC-6 (Least Privilege) guidance
- Developing recommendations to improve access governance
- Supporting privacy and data protection objectives

---

## Key Findings

The assessment identified several control weaknesses:

### Excessive Permissions

- Internal folder permissions were not sufficiently restricted
- Users possessed access rights beyond business requirements
- Access governance controls were ineffective

### Unauthorized Information Exposure

- Sensitive information was unintentionally shared
- Access was granted without a legitimate business need
- Data exposure risk increased significantly

### Lack of Privilege Reviews

- No regular access review process existed
- Temporary permissions were not consistently removed
- Excessive privileges accumulated over time

---

## Recommendations

### Role-Based Access Control (RBAC)

Implement role-based access controls to ensure users receive only the permissions required to perform their job functions.

### Least Privilege Enforcement

Restrict access to sensitive resources based on business need and approved authorization.

### Access Reviews and Auditing

- Conduct periodic access reviews
- Perform privilege audits
- Validate user permissions regularly

### Temporary Access Management

Automatically revoke temporary permissions when no longer required.

### Activity Monitoring

Maintain audit logs of privileged user activity to improve accountability and support future investigations.

Successfully analyzed a data leakage incident caused by excessive permissions and weak access governance practices.
