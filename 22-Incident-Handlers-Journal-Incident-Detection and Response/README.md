# Incident Handler’s Journal – Incident Detection and Response

## Project Overview

This project focused on documenting and analyzing multiple cybersecurity incident response scenarios using an Incident Handler’s Journal.

Through a series of investigative activities, I practiced recording incident details, identifying indicators of compromise (IoCs), analyzing network traffic, investigating malicious file hashes, and documenting findings using industry-standard incident response methodologies.

The project emphasized the importance of structured documentation, repeatable investigation processes, and effective incident response practices used within Security Operations Centers (SOC).

---

## Objective

The purpose of this project was to:

- Practice cybersecurity incident documentation techniques
- Analyze security events and indicators of compromise
- Investigate suspicious files and network activity
- Apply incident response lifecycle concepts
- Use security analysis tools during investigations
- Improve analytical and technical reporting skills

---

## Key Activities

### Incident Documentation

- Documented a ransomware incident using the 5 W’s framework:
  - Who
  - What
  - When
  - Where
  - Why

- Recorded observations, findings, lessons learned, and investigative notes within an Incident Handler’s Journal.

### Network Traffic Analysis

- Analyzed packet capture (PCAP) files using Wireshark
- Investigated network communications and traffic patterns
- Identified potentially suspicious network activity

### Traffic Capture and Filtering

- Captured and filtered network traffic using tcpdump
- Applied command-line packet analysis techniques
- Reviewed traffic data for investigation purposes

### Malware and IoC Investigation

- Investigated suspicious SHA-256 file hashes
- Used VirusTotal to validate malicious indicators
- Identified and documented indicators of compromise (IoCs)

### Incident Response Lifecycle

Applied incident response phases including:

1. Detection
2. Analysis
3. Containment
4. Eradication
5. Recovery

### Professional Reflection

- Reflected on investigation challenges
- Evaluated learning progress
- Identified opportunities to improve security awareness and response effectiveness
