# Network Security Incident Analysis Using NIST CSF

## Overview

This project presents a simulated network security incident involving a denial-of-service (DoS) attack that disrupted network operations and prevented users from accessing critical services.

The incident was analyzed using the NIST Cybersecurity Framework (CSF) to evaluate response activities across the Identify, Protect, Detect, Respond, and Recover functions.

## Incident Summary

The organization experienced a denial-of-service attack caused by a flood of ICMP packets that overwhelmed network resources and disrupted normal operations.

Investigation revealed that an improperly configured firewall allowed excessive ICMP traffic into the environment, creating a vulnerability that enabled the attack.

The cybersecurity team responded by blocking malicious traffic, restoring services, and implementing additional security controls to improve resilience.

---

## NIST Cybersecurity Framework Analysis

### Identify

The investigation determined:

- ICMP flood activity targeting internal network resources
- Service availability disruptions
- Excessive inbound ICMP traffic
- Firewall configuration weaknesses
- Increased exposure to denial-of-service attacks

### Protect

To reduce future risk, the organization implemented:

- Improved firewall rule configurations
- ICMP rate-limiting controls
- Additional network security protections
- IDS/IPS deployment
- Enhanced traffic filtering mechanisms

### Detect

Detection capabilities focused on:

- Network traffic monitoring
- Identification of abnormal ICMP activity
- Detection of suspicious traffic patterns
- Security event analysis
- Monitoring for denial-of-service indicators

### Respond

Response activities included:

- Blocking malicious ICMP traffic
- Isolating affected services
- Prioritizing restoration of critical systems
- Investigating attack vectors
- Documenting incident findings

### Recover

Recovery efforts focused on:

- Restoring normal network operations
- Returning critical services to production
- Re-establishing business operations
- Improving monitoring capabilities
- Strengthening future resilience

---

## Key Findings

The investigation identified:

- Improper firewall configuration
- Insufficient traffic filtering controls
- Lack of effective ICMP rate limiting
- Increased exposure to denial-of-service attacks
- Need for enhanced monitoring and detection capabilities

---

## Lessons Learned

The incident highlighted the importance of:

- Proper firewall configuration management
- Continuous network monitoring
- Layered security controls
- Effective incident response procedures
- Proactive traffic analysis
- Regular security control reviews

---

## Security Improvements

Recommended improvements included:

- Enhanced firewall rule management
- ICMP rate limiting
- IDS/IPS deployment
- Continuous traffic monitoring
- Improved incident response procedures
- Periodic security configuration reviews
